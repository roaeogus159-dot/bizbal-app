## 4. 실행·배포·캡처 방법
- **실행**: `npm install` → `npm run dev` (또는 `.claude/launch.json`의 bizbal-dev, 포트 5199)
- **빌드**: `npm run build` → dist/
- **배포**: GitHub push → Vercel Import → Deploy (설명서 3장에 단계별 정리)
- **화면 캡처 재생성**: dev 서버 켠 상태에서 `node Reference/capture.mjs`
- **데모 URL**: `?demo=convert|editor|result|library` → 샘플 자동 로드 후 해당 화면 이동 (검증·캡처용)
- **설명서 재생성**: 스크래치패드의 make_manual.py 참고 (스크린샷 → base64 임베드 HTML)

## 5. 설계 결정·주의사항
- **색 연산을 culori 대신 직접 구현**: 핫루프(수만 칸 × 85색 ΔE2000)에서 객체 할당 없는 순수 숫자 함수가 필요했음. `lib/color.ts`에 Sharma 기준 CIEDE2000. 검증된 표준 공식이며 워커에서 RGB 캐시와 함께 사용.
- **grid는 전체 팔레트 인덱스(Uint16)**: 팔레트 배열 = 기본 85색 + 커스텀(추가 순). **커스텀 색 삭제 시 자리 유지(deleted 플래그)** — 인덱스가 밀리면 기존 grid가 깨지기 때문. UI에서만 숨김.
- **미리보기 성능**: 칸<2.5px 또는 가시 칸>4만이면 1칸=1px 오버뷰 비트맵을 스케일 드로우, 그 외엔 (색×재질×크기) 스프라이트 프리렌더 후 스탬프.
- **재변환 시 수동 수정 소실** → 크기/팔레트 변경 전 window.confirm 경고 (plan.md 3.3 정책).
- **자동저장**: 20만 칸 초과 grid는 저장 안 함(localStorage 5MB 한계). 복원 시 cellRgb/deltaE 없음 → 전문가 강조는 재변환 후 표시됨.
- **iOS 저장**: navigator.canShare({files}) 지원 시 공유시트(사진 저장), 아니면 다운로드 폴백(350ms 간격 순차).
- **알려진 버그 수정 이력**: decodeImage에서 bmp.close() 후 bmp.width 참조로 aspect가 항상 1이 되던 버그 수정됨 (convert.ts).
